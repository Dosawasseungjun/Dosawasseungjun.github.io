# FI 모듈(Financial Accounting 재무회계)

## FI 모듈이 하는 일

> **"회사에서 일어나는 모든 돈의 흐름을 기록하고 관리하는 시스템"**

회사의 가계부를 잘 작성하는 것이 FI 모듈의 핵심 업무다.

- 여러 회사(법인)의 돈을 동시에 관리한다.
- 모든 거래를 전표(Document)라는 단위로 기록한다.
- 채권, 채무를 체계적으로 추적한다. 


```
💡 전표(Document)란?

"언제, 어떤 계정으로, 얼마가 들어오고 나갔는지"를 기록한 회계 증빙 단위

예시) 2026.03.24 / 이룸가구 / 거래처 A에 가구 납품 / 매출 500만원 발생
```

## 회사 코드 (Company Code)

| 회사코드 | 회사명     | 특징           |
| ---- | ------- | ------------ |
| IRUM | (주)이룸   | 본사           |
| IRUG | (주)이룸가구 | 가구 제조/판매 자회사 |
| IRUR | (주)이룸렌탈 | 렌탈 서비스 자회사   |

위 처럼 각각의 법인을 회사코드로 관리한다. 
모든 재무 데이터는 이 회사 코드 단위로 기록되고 분리된다. 

## 계정과목
모든 돈의 흐름은 계정과목이라는 분류 코드로 구분한다. 

```
자산  ├─ 유동자산
      │   ├─ 현금및현금성자산  (계정코드: 1001)
      │   ├─ 매출채권          (계정코드: 1101)
      │   └─ 미수금            (계정코드: 1102)
      └─ 비유동자산
          └─ 유형자산          (계정코드: 1201)

부채  ├─ 매입채무              (계정코드: 2001)
      └─ 미지급금              (계정코드: 2002)

수익  └─ 매출액               (계정코드: 4001)

비용  └─ 매출원가             (계정코드: 5001)
```

재무제표(B/S = Balance Sheet)
손익계산서(P&L = Profit and Loss Statement)를 잘 작성하려면 계정과목표를 잘 만들어둬야할 것이다. 

## 채권(AR, Accounts Receivable)

> 채권 = **고객이 우리에게 갚아야 할 돈**

|채권 유형|설명|예시|
|---|---|---|
|매출채권|물건/서비스를 팔고 아직 못 받은 돈|가구 납품 후 미수금|
|미수금|영업 외 활동으로 발생한 미회수금|비품 처분 후 미회수|
|기타채권|그 외 분류|직원 가지급금 등|

## 여신(Credit Management)

> 여신 = **고객에게 얼마까지 외상을 허용할 것인가의 한도**

```
예시)
  거래처 A마트의 여신 한도: 1억원
  현재 외상 사용액:         7,000만원
  남은 여신 한도:           3,000만원
  
  → 만약 A마트가 추가로 5,000만원 주문을 넣는다면?
    한도 초과! → 거래 차단 또는 경고 발생
```

- 여신 관리를 못하면?
	- 외상을 너무 많이 줬다가 못 받는 상황이 생긴다.

```
💡 대손충당금이란?
   "이 외상은 못 받을 수도 있으니 미리 손실로 잡아두자"는 회계 처리
   
   연체 90일 이상 → 대손충당금 설정률 50% 적용
   → 회사는 받을 돈의 절반을 이미 손실로 회계 처리
```

## 데이터 흐름
```
전표 입력 (FI 전표 생성)
        │
        ▼
전표 Header & Item 저장
        │
        ▼
채권현황 테이블 업데이트 ◄── 여신(Credit Management) 테이블
        │
        ▼
채권 & 여신 통합 조회 화면
        │
        ├── 채권 미회수 잔액 계산 (발생 - 회수)
        ├── 연체일수 자동 계산
        ├── 여신 한도 대비 사용률 계산
        └── 대손충당금 자동 산정
```

## 전표 (Document)

### 전표의 이해
```
┌─────────────────────────────┐
│  GS25 홍대점                 │  ← 어느 가게 (= 회사코드)
│  2026.03.24  14:32           │  ← 언제 (= 증빙일/전기일)
│  영수증번호: 00012345         │  ← 고유 번호 (= 전표번호)
├─────────────────────────────┤
│  콜라          1,500원       │  ← 품목 1 (= 아이템 1번)
│  과자          2,000원       │  ← 품목 2 (= 아이템 2번)
├─────────────────────────────┤
│  합계          3,500원       │
└─────────────────────────────┘
```

- 영수증 상단 정보 -> 전표 Header
- 영수증 품목 목록 -> 전표 Item

### 전표의 구조

```
┌──────────────────────────────────────────────────────┐
│                   전표 HEADER                         │
│  전표번호: 0100000001  │  회사: IRUM  │  연도: 2026   │
│  전표유형: SA          │  전기일: 2026.03.24           │
│  통화: KRW             │  헤더텍스트: 3월 임대료 지급  │
└──────────────────────────────────────────────────────┘
                    │ 1개의 Header
                    │ N개의 Item
┌──────────────────────────────────────────────────────┐
│  Item 001 │ 차변(S) │ 계정: 임대료(비용) │ 500,000원  │
│  Item 002 │ 대변(H) │ 계정: 보통예금     │ 500,000원  │
└──────────────────────────────────────────────────────┘
```

> 차변 합계 = 대변 합계

위 규칙이 틀리면 전표가 생성이 되지 않는다. 

### 차변(Debit)과 대변(Credit)

> 모든 거래는 두 가지 계정에 동시에 영향을 준다. 

```
차변 (Debit, S)              대변 (Credit, H)
       ─────────────────            ─────────────────
       자산이 늘어날 때       vs    자산이 줄어들 때
       비용이 발생할 때       vs    수익이 발생할 때
       부채가 줄어들 때       vs    부채가 늘어날 때
```

#### 예시 1 : 가구를 팔고 현금을 받음 (매출 발생)
이룸가구가 A호텔에 가구 100만원어치를 팔고 현금을 받은 상황

| Item | 차대변    | 계정         | 금액        |
| ---- | ------ | ---------- | --------- |
| 001  | S (차변) | 현금 및 현금성자산 | 1,000,000 |
| 002  | H (대변) | 매출액(수익 발생) | 1,000,000 |

#### 예시 2 : 가구를 팔았는데 나중에 받기로 함 (외상 매출)
채권이 생김.

| Item | 차대변    | 계정          | 금액        |
| ---- | ------ | ----------- | --------- |
| 001  | S (차변) | 매출채권(자산 증가) | 1,000,000 |
| 002  | H (대변) | 매출액 (수익 발생) | 1,000,000 |

다음 달, A 호텔이 현금을 입금했다면? 
**채권이 해소된다.**

| Item | 차대변    | 계정                | 금액        |
| ---- | ------ | ----------------- | --------- |
| 001  | S (차변) | 현금 및 현금성자산(자산 증가) | 1,000,000 |
| 002  | H (대변) | 매출채권(자산 감소)       | 1,000,000 |

#### 예시 3 : 임대료를 현금으로 지급함 (비용 발생)
가구 제조 기계의 임대료를 내는 과정이라고 생각해보자. 

| Item | 차대변    | 계정    | 금액      |
| ---- | ------ | ----- | ------- |
| 001  | S (차변) | 임대료비용 | 500,000 |
| 002  | H (대변) | 보통예금  | 500,000 |

### 역분개(Reversal)

> 전표를 잘못 입력했을 때, 그냥 삭제하는 것이 아니라 반대 방향으로 새 전표를 만들어 효과를 상쇄한다. 

```
원본 전표 (0100000001):
  Item 001 │ 차변 │ 임대료비용  500,000
  Item 002 │ 대변 │ 보통예금    500,000

역분개 전표 (0100000002):
  Item 001 │ 대변 │ 임대료비용  500,000  ← 방향이 반대!
  Item 002 │ 차변 │ 보통예금    500,000  ← 방향이 반대!
```

### 전표 생성 과정

```
[1단계] 거래 발생
        ↓
        예) 이룸가구가 B편의점에 가구 200만원 납품 완료

[2단계] 전표 입력 (or 자동 생성)
        ↓
        사용자가 SAP에 전표 정보 입력
        또는 다른 모듈(SD, MM, HR)에서 자동으로 전표 생성

[3단계] 전표 검증 (SAP 자동)
        ↓
        ✅ 차변 합계 = 대변 합계 ?
        ✅ 계정코드 유효한가?
        ✅ 회사코드 유효한가?
        ✅ 여신 한도 초과 여부?

[4단계] 전표 저장 (전기, Posting)
        ↓
        전표번호 자동 채번 (예: 0100000001)
        전표 Header 테이블 저장
        전표 Item 테이블 저장 (라인별)
        채권현황 테이블 업데이트

[5단계] 관련 모듈로 데이터 전파
        ↓
        CO: 코스트센터/프로핏센터별 비용-수익 반영
        AR: 채권 잔액 업데이트
        Credit: 여신 사용액 업데이트
```

### 다른 모듈과의 연결 간단 정리
![모듈 간 연결](/assets/image/module.png)

```
[SD: 수주/납품]      [MM: 구매/입고]      [HR: 급여]
           │                    │                  │
           └────────────────────┼──────────────────┘
                                │ 자동 전표 생성
                                ▼
                    ┌───────────────────────┐
                    │       FI 전표          │
                    │  ┌─────────────────┐  │
                    │  │  Header         │  │
                    │  │  (전표번호/날짜)  │  │
                    │  └────────┬────────┘  │
                    │           │ 1:N        │
                    │  ┌────────┴────────┐  │
                    │  │  Item 001 (차변) │  │
                    │  │  Item 002 (대변) │  │
                    │  │  ...            │  │
                    │  └────────┬────────┘  │
                    └───────────┼───────────┘
                                │
              ┌─────────────────┼─────────────────┐
              ▼                 ▼                  ▼
        [채권현황]          [여신관리]            [CO 모듈]
      채권 발생/회수       여신 사용액           코스트/프로핏
        잔액 추적           한도 관리            센터 비용 배분
```
